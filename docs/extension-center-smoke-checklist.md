# Проверка центра расширений

Документ описывает ручную проверку центра расширений после изменений источников, каталога, Markdown-карточек, установки пакетов или diagnostics.

## Источники

- Открыть страницу дополнений и проверить, что на ней не отображается экранная клавиатура.
- Открыть модальное окно источников и проверить доступность форм для локального источника, URL и прямой ссылки.
- Добавить или изменить локальный источник и проверить карточку: имя, версия, иконка, описание, путь, количество пакетов, статус синхронизации и время последней синхронизации.
- Отключить и снова включить источник; проверить переход записей каталога между disabled и active состояниями.
- Синхронизировать корректный источник и проверить обновление каталога без stale-дубликатов.
- Синхронизировать источник с некорректной Markdown-карточкой и проверить, что источник остаётся доступным, а запись показывает предупреждение.

## Каталог

- Проверить фильтры каталога: все пакеты, установленные пакеты, аддоны, модификации и темы.
- Проверить сортировку: доступные обновления и устанавливаемые пакеты должны быть выше заблокированных, manual-only, invalid и diagnostic duplicate entries.
- Проверить поиск по имени, id, имени источника, автору и описанию.
- Проверить, что incompatible, manual-only, blocked dependency, broken card и duplicate recommendation entries отображаются с понятной причиной.
- Проверить компактное отображение duplicate source text, если один пакет предоставляют больше трёх источников.
- Проверить, что duplicate recommendations выбирают предпочтительный доступный источник, если самая новая версия заблокирована.

## Markdown-карточки

- Нажать на карточку каталога с валидным Markdown-описанием и проверить открытие модального окна по нажатию на карточку.
- Проверить поддержку заголовков, ссылок, изображений, жирного текста и курсива.
- Проверить отображение автора, версии, минимальной версии приложения, совместимости, issues и кнопки установки.
- Установить или обновить пакет из модального окна и проверить изменение состояния кнопки без ручной перезагрузки.
- Нажать на карточку без валидного Markdown и проверить, что пустое модальное окно не открывается.

## Установка

- Установить аддон из локального источника и проверить появление в установленных аддонах.
- Установить тему из локального источника и проверить появление в установленных темах и доступность выбора.
- Установить локальный mod package и проверить копирование manifest, entry script, files и locales.
- Обновить установленный аддон, модификацию и тему с версии 1 до версии 2 и проверить состояние `installed`.
- Проверить, что удалённые модификации без `manifest.files` остаются manual-only.
- Проверить, что модификации с отсутствующими runtime dependencies блокируются до установки.
- Проверить, что несовместимые пакеты блокируются, а несовместимые обновления показывают issue установки.

## Диагностика

После изменений центра расширений запускать:

```bash
npm run diagnostics:extensions
npm run diagnostics:i18n
npm run build
```
